# Backend API Contracts

This folder defines endpoint-shaped contracts for the future Warframe assistant backend.

These files are not server code yet. They describe stable request and response shapes that the app can target later while this knowledge base keeps evolving underneath.

Current endpoints:

- `POST /assistant/query`: route a natural-language question and return an assistant-ready payload.
- `GET /player/inventory-summary`: return flattened owned inventory for a player.
- `GET /player/live-context`: return one fused low-latency payload for the overlay assistant.
- `GET /player/live-context/poll`: return live context only when its change token differs from the caller's last token.
- `POST /overlay/inventory-sync`: accept overlay/OCR inventory snapshots and return normalized merge results.
- `POST /overlay/loadout-sync`: accept overlay/app current loadout and upgrade-state snapshots for session/build merge previews.
- `POST /overlay/mission-sync`: accept overlay/app current mission context snapshots for session merge previews.
- `POST /overlay/event-feed`: append overlay/OCR gameplay events such as pickups, objective progress, mission phases, and player-state warnings.

Overlay writes are preview-first. Persistent writes require `writeMode: "persistent"`, `confirmWrite: true`, and an `x-warframe-kb-api-key` header matching `WARFRAME_KB_API_KEY`; the local API writes a backup before changing player inventory, session, or build-template state.

Confirmed overlay writes also attempt a lightweight live-context refresh. Responses include `liveContextRefresh.status`, which is `refreshed` for supported materialized players, `skipped` for unsupported player views, or `failed` when the write succeeded but the refresh command did not.
